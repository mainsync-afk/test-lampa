# trakt_folder_sync

Плагин для Lampa. Синхронизация папок Избранного с Trakt.tv.

**Trakt — единственный источник правды.** Lampa показывает то, что сейчас в Trakt, и ничего своего не хранит. Действия пользователя в Lampa транслируются в Trakt API. Полная спецификация поведения — в [SPEC.md](SPEC.md).

## Установка

Постоянная ссылка для Lampa (всегда указывает на актуальную версию):

```
https://mainsync-afk.github.io/test-lampa/trakt_folder_sync.js
```

Для обхода кэша после обновления можно добавлять `?v=N` — `?v=1`, `?v=2` и т.д.

## Папки

Плагин работает с пятью папками Lampa, отражающими состояние Trakt:

| Папка Lampa | Источник в Trakt | Направление |
|---|---|---|
| Закладки (book) | Watchlist | Двусторонняя |
| Смотрю (look) | Сериалы с непросмотренными вышедшими эпизодами | Двусторонняя |
| Просмотрено (viewed) | История просмотра + полностью закрытые сериалы | Двусторонняя |
| Продолжение следует (continued) | Сериал догнан, но ещё выходит | Trakt → Lampa |
| Брошено (thrown) | Триплет: `/users/hidden/progress_watched` (для MB) + `/users/hidden/dropped` (для Trakt-веб) + кастомный user list (для визуального маркера). Читается как union | Двусторонняя |

Логика статусов сериалов — в [SPEC.md §«Логика статусов сериала»](SPEC.md#логика-статусов-сериала).

## Настройки

Плагин добавляет свои переключатели в существующий раздел **Trakt** (от плагина `trakt_by_lampame` / `trakttv`):

- **Синхронизация папок с Trakt (v\<версия\>)** — общий выключатель.
- **Окно Pending Ops** (с 0.8.3) — 15 мин / 5 мин / 1 мин / 10 сек / Выключено. По умолчанию 15 мин: буфер защищает только что изменённые карточки от отката, пока Trakt API переливает запись. Меньше — быстрее тестировать. «Выключено» полностью отключает буфер.

Переключатель логов (`trakt_enable_logging`) управляется через этот же раздел от `trakt_by_lampame`.

Своего раздела в панели настроек плагин не заводит: проба в v0.4.0–0.4.1 показала, что `Lampa.SettingsApi.addComponent` не всегда приводит к появлению раздела в UI даже при успешной регистрации. Паттерн v44 (добавление параметров в чужой компонент) работает надёжно.

## Зависимости

Плагин `trakt_by_lampame` должен быть установлен и авторизован в Trakt. Берём из него: OAuth-токен (в `Lampa.Storage['trakt_token']`), scrobbling эпизодов, UI логина. Данные с Trakt читаем и отправляем сами — через общий прокси `apx.lme.isroot.in/trakt`, который подставляет нужный client_id.

После Promise.all per-folder синхронизаций — финальный verify-проход для страховки.

С 0.8.7 ищем настоящую карточку через `findRealCard` по всем известным папкам Lampa (book/like/wath/history/look/viewed/continued/thrown/scheduled/card) и передаём её. - [x] **Синхронизация отметок просмотра серий** (0.9.0 → 0.9.1): три направления — отметить серию в Lampa → отметится в Trakt (работало и раньше через scrobble в `trakt_by_lampame`), снять в Lampa → снимется в Trakt (раньше было no-op, ломалось при ручной отметке «не смотрел»), отметить/снять в Trakt → подтянется в Lampa (раньше Lampa жила своей жизнью). Trakt — источник правды: при открытии Избранного тянем `/sync/watched/shows` (TMDB→Trakt id mapping), затем для каждого сериала, у которого в Trakt есть хоть один просмотренный эпизод — `/shows/:trakt_id/progress/watched` и сверяем с `Lampa.Timeline` (storage `file_view`, ключ — `Utils.hash([season, sep, episode, original_name||original_title])`). Чего нет в Lampa, но есть в Trakt — пишем `Timeline.update({percent:100, received:true, ...})`. Чего нет в Trakt, но есть в Lampa с percent ≥ 90 — оставляем (уже отмеченное юзером не трогаем; реальная отметка прилетит в Trakt от scrobble). На стороне Lampa слушаем `Timeline.listener('update')`: percent=0 → `POST /sync/history/remove` с `{shows:[{ids:{tmdb}, seasons:[{number, episodes:[{number}]}]}]}` (фикс «снял в Lampa — осталось в Trakt»). Защита от петли — два независимых флага: `_selfWriteMark[hash]` ставит наш `markEpisodeInLampa` (listener percent=0 трактует как echo и игнорирует), `_pendingLampaMark[hash]` ставит listener при percent ≥ 90 (используется только в reconcile, чтобы не снять отметку до того, как chuck-плагин допишет её в Trakt). Сделано простым pull-reconcile (а не realtime push): по требованию пользователя — «менее нагрузочно для системы, отметки серий не суперважная штука, потеря раз в месяц не страшна». В 0.9.0 было два бага, исправлены в 0.9.1: (1) `_pendingLampaMark` использовался и в listener, и в reconcile, и при тапе по серии «отметил → percent=95 → ставит pending → тап «снял» → percent=0 → блокируется тем же pending'ом» — listener не пускал legitimate user-unmark в Trakt. Разделил на два флага. (2) `/shows/:id/progress/watched` ожидает Trakt id, а не TMDB. С TMDB id Trakt отдавал прогресс случайного сериала с тем же числом — `e.completed` везде false, ничего не помечалось. Резолвим TMDB→Trakt через mapping из `/sync/watched/shows`, шоу без watched в Trakt пропускаем (там нечего синхронизировать).
- [x] **Защита ручного снятия серии от reconcile** (0.9.3): добавили симметричный к `_pendingLampaMark` буфер `_pendingLampaUnmark[hash]` на 60 сек. До 0.9.3: юзер снимал отметку с серии в Лампе → плагин слал `POST /sync/history/remove` мгновенно, но Trakt применял запись с задержкой 5–15 мин (stale-окно). За это окно успевал прокрутиться reconcile, видел «в Trakt смотрена / в Lampa нет» и перепомечал серию обратно — пользователь видел flicker «снял → вернулась → снова снялась». Защита `_pendingLampaMark` (0.9.1) работала только для отметки (percent ≥ 90), снятие было не защищено — это родовой пробел дизайна. Теперь listener при percent=0 (после прохождения проверки `isSelfWriteMark`) ставит `_pendingLampaUnmark[hash] = Date.now()`, и reconcile в ветке `traktWatched && !lampaWatched` проверяет флаг и пропускает перепометку. На write-путь к Trakt не влияет — `/sync/history/remove` уходит как раньше.
- [x] **Брошено: триплет ячеек Trakt + read-union + write-канонизация** (0.10.0): полный clean-rewrite модуля Брошено по спеке. Trakt не имеет одного «правильного» поля «брошено» — разные клиенты пишут в разные структуры: Moviebase «Stop watching» → `/users/hidden/progress_watched`, Trakt-веб «Drop show» → `/users/hidden/dropped`, кастомный список нужен как визуальный маркер на постерах в UI MB / Trakt-веб. Чтобы любой клиент при подключении понял «сериал брошен», плагин пишет одновременно в три ячейки и читает union. Идентификация кастомного списка — по `ids.trakt` (числовой, стабилен), а не по slug (slug ломается при ручном переименовании списка). При запуске настроек подгружается `/users/me/lists` для селектора, кешируется в Storage. Если список не выбран — пишем только в hidden-ячейки (визуальный маркер не появится). Read-union: parallel GET трёх ячеек, union по tmdb id, **canonization** — для каждой карточки в union, которая есть не во всех трёх ячейках, fire-and-forget POST'ом дописываем недостающие (это лечит действия других клиентов: MB пишет только в hpw, Trakt-веб только в hdr). Классификатор-приоритет: thrown → look/viewed/continued. Для фильмов: hidden-эндпоинты не поддерживают `type=movie`, поэтому фильмы попадают только в кастомный список; без настроенного списка фильм в Брошено локально не удержится. Pending Op закрывает stale-окно Trakt (5–15 мин). Архитектура подробно — в SPEC.md «Логика Брошено».

- [x] **Брошено: расширенное логирование канонизации** (0.10.0-rc4-diag): диагностический build для расследования кейсов «карточка попала в папку Брошено в одном клиенте, не появилась в скрытых другого». Расширили четыре места: (1) `readThrownUnion` теперь печатает per-card `{tmdb, trakt, title, hpw, hdr, list}` — видно, в каких ячейках Trakt сейчас сидит каждый id из union. (2) `writeCanonizationThrown` печатает планируемые операции с `tmdb/trakt/title/type` — видно, что именно собираемся дописать. (3) `postHiddenAdd`/`postHiddenRemove` логируют отправленный body и поля ответа Trakt `added/deleted/not_found` — видно, что Trakt принял, что вернул в not_found. (4) `postListAdd`/`postListRemove` — то же с `existing`. Цель — увидеть, какая именно карточка persistently отказывается записаться в `/users/hidden/dropped` несмотря на 201, и что Trakt говорит про неё в `not_found`.
- [x] **Брошено: фикс «THE PITT bug» — двойной дедуп пропадал add/remove** (0.10.0-rc3): на rc2 переброс из Смотрю в Брошено в Lampa коротко проставлял Брошено, потом сбрасывал, и сериал возвращался обратно в Смотрю. Причина: `pushHistory` в самом начале записывает ключ `(action, folder, id)` в `_recentWrites` (защита от двойного эмита Lampa), затем диспатчит в `addThrown`/`removeThrown`. А внутри `addThrown`/`removeThrown` стоял ещё один `checkWriteDedup` — он попадал на свой же только что записанный ключ, возвращал `true`, и весь add/remove отваливался: ни один POST в Trakt не уходил, Pending Op для thrown не записывался. На следующий reconcile Pending Op `remove look` чистил id из всех желаемых статусных папок (включая thrown), а после истечения TTL Trakt всё ещё показывал сериал в `/sync/watched/shows` — классификатор возвращал его в Смотрю. Лечится снятием redundant-проверки внутри `addThrown`/`removeThrown` — `pushHistory` уже отдедупил.

- [x] **Эксклюзивность статусных папок Лампы расширена на scheduled/thrown** (0.9.2): до этого `enforceStatusExclusivity` и `verifyStatusFoldersClean` снимали конфликтующие mark только из `STATUS_FOLDERS = ['look','viewed','continued']` — папок, которые плагин read/write-ит к Trakt. Если юзер вручную ставил «Запланировано» (`scheduled`) или «Брошено» (`thrown`) на сериал, плагин потом классифицировал его в look/viewed/continued, ставил свой mark — но scheduled/thrown оставались. В UI Лампы это «двойной статус», логически противоречивый (нельзя одновременно «Запланировано» и «Просмотрено»). Завели вторую константу `LAMPA_STATUS_FOLDERS = ['look','viewed','continued','scheduled','thrown']` — полный набор взаимно-исключающих статусных папок Лампы. `STATUS_FOLDERS` остался для read/write путей синхронизации с Trakt (плагин этими папками реально обменивается данными), а внутренний цикл exclusivity бегает по `LAMPA_STATUS_FOLDERS` и снимает mark с любой статусной папки Лампы кроме той, которую плагин решил оставить. Воспроизведение бага — лог `lampa.mx-1777192086909.log` (A Dream Within a Dream, id 253193, отметка серии в Лампе) и `lampa.mx-1777192499438.log` (Euphoria, id 85552, отметка серии в Trakt) на v0.9.1.

Вне скоупа (решено): Нравится, Позже, Запланировано, История просмотров, оценки.
