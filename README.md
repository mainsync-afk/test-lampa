# trakt_folder_sync

Плагин для Lampa. Синхронизация папок Избранного с Trakt.tv.

**Trakt — единственный источник правды.** Lampa показывает то, что сейчас в Trakt, и ничего своего не хранит. Действия пользователя в Lampa транслируются в Trakt API. Полная спецификация поведения — в [SPEC.md](SPEC.md).

## Установка

Постоянная ссылка для Lampa (всегда указывает на актуальную версию):

```
https://mainsync-afk.github.io/test-lampa/trakt_folder_sync.js
```

Для обхода кэша после обновления можно добавлять `?v=N` — `?v=1`, `?v=2` и т.д.

## Папки

Плагин работает с пятью папками Lampa, отражающими состояние Trakt:

| Папка Lampa | Источник в Trakt | Направление |
|---|---|---|
| Закладки (book) | Watchlist | Двусторонняя |
| Смотрю (look) | Сериалы с непросмотренными вышедшими эпизодами | Двусторонняя |
| Просмотрено (viewed) | История просмотра + полностью закрытые сериалы | Двусторонняя |
| Продолжение следует (continued) | Сериал догнан, но ещё выходит | Trakt → Lampa |
| Брошено (thrown) | Кастомный Trakt user list «Брошено» + `/users/hidden/dropped` как helper | Двусторонняя |

Логика статусов сериалов — в [SPEC.md §«Логика статусов сериала»](SPEC.md#логика-статусов-сериала).

## Настройки

Плагин добавляет свои переключатели в существующий раздел **Trakt** (от плагина `trakt_by_lampame` / `trakttv`):

- **Синхронизация папок с Trakt (v\<версия\>)** — общий выключатель.
- **Окно Pending Ops** (с 0.8.3) — 15 мин / 5 мин / 1 мин / 10 сек / Выключено. По умолчанию 15 мин: буфер защищает только что изменённые карточки от отката, пока Trakt API переливает запись. Меньше — быстрее тестировать. «Выключено» полностью отключает буфер.

Переключатель логов (`trakt_enable_logging`) управляется через этот же раздел от `trakt_by_lampame`.

Своего раздела в панели настроек плагин не заводит: проба в v0.4.0–0.4.1 показала, что `Lampa.SettingsApi.addComponent` не всегда приводит к появлению раздела в UI даже при успешной регистрации. Паттерн v44 (добавление параметров в чужой компонент) работает надёжно.

## Зависимости

Плагин `trakt_by_lampame` должен быть установлен и авторизован в Trakt. Берём из него: OAuth-токен (в `Lampa.Storage['trakt_token']`), scrobbling эпизодов, UI логина. Данные с Trakt читаем и отправляем сами — через общий прокси `apx.lme.isroot.in/trakt`, который подставляет нужный client_id.

- [x] Собственный сетевой слой `traktFetch` к прокси Trakt.
- [x] Синхронизация папки **Закладки** из Trakt Watchlist при открытии экрана Избранного.
- [x] Переключатель в существующем разделе «Trakt» (свой компонент не создаём — Lampa не подхватывала addComponent в UI).
- [x] Обогащение карточек данными TMDB (постеры, локализация, overview).
- [x] **Write-путь для Закладок:** добавление/удаление в Lampa → `POST /sync/watchlist[/remove]` в Trakt. Защита от петли — `_ownOps` с TTL-окном и дедуп исходящих запросов (Lampa иногда эмитит события парно).
- [x] **Pending Ops** (TTL 15 мин): только что добавленный фильм не исчезает из папки, пока Trakt API переливает изменение.
- [x] **Надёжное определение типа:** до 0.8.3 в `/sync/watchlist` шли id одновременно в `movies` и `shows`, фактический тип читался из `added`/`deleted` в ответе Trakt. С 0.8.4 переехали на резолв типа до отправки через `resolveCardType` (поля карточки → fallback `/search/tmdb/:id`) — см. ниже фикс «movie+show с одним tmdb id».
- [x] **Статусные папки — read-путь** (0.6.0+): Смотрю (look), Просмотрено (viewed), Продолжение следует (continued). Список сериалов — `/sync/watched/shows?extended=full`, на каждый сериал — `/shows/:id/progress/watched` за `next_episode`. Фильмы в viewed — `/sync/watched/movies`.
- [x] **Классификация по `next_episode`** (0.7.1): решающий сигнал догнанности — `progress.next_episode == null`. Это ровно тот же сигнал, по которому Trakt UI показывает/не показывает сериал в Continue Watching. Бесплатная замена VIP-эндпоинта `up_next_nitro`. В 0.7.0 был промежуточный вариант по `last_episode` — он оказался неверным (в этом эндпоинте `last_episode` = последняя *просмотренная*, а не последняя вышедшая).
- [x] **Локализация заголовков сериалов в list-view папок** (0.6.1): до этого Trakt-title (английский) затмевал TMDB-name (локализованный).
- [x] **Статусные папки — write-путь** (0.8.0): действия пользователя транслируются в `POST /sync/history[/remove]`. Фильм → Просмотрено — по tmdb id. Сериал → Просмотрено — резолвим trakt id (по trakt/imdb/`/search/tmdb`), тянем progress, шлём одним запросом все вышедшие эпизоды из `seasons[].episodes[]`. Сериал → Смотрю — помечаем только S01E01 как «подсказку» Trakt'у (completed становится > 0, `next_episode` указывает дальше, классификатор отдаёт look). Обратные действия зеркальны. Убрать из Смотрю / Фильм → Смотрю / любое → Продолжение следует — no-op (производные состояния). Pending Ops для статусных папок: `add` в папку X чистит id из остальных статусных desired, `remove` чистит id из всех статусных desired — пока Trakt переливает запись, карточка не «уезжает» обратно.
- [x] **Статусные папки — thin-card fix** (0.8.1): Lampa в листенере часто отдаёт тощую карточку без `method`/`card_type`/`number_of_seasons` — из-за этого Breaking Bad/Fallout принимались за фильмы и write-путь у look молча игнорировал их. Теперь для look безусловно трактуем как сериал (look — show-only в Trakt), для viewed резолвим тип через `/search/tmdb/:id`, а не по полям карточки. Тот же паттерн, что был применён для book в 0.5.2.
- [x] **Remove из look — зеркало add** (0.8.2): до 0.8.1 удаление из Смотрю было no-op, из-за чего добавленный через нашу же подсказку S01E01 залипал в Trakt и сериал возвращался в look на следующей синхронизации. Теперь remove постит `POST /sync/history/remove` на тот же S01E01. Реальную историю за пределами S01E01 не трогаем — если она есть, классификатор корректно оставит сериал в look/viewed/continued.
- [x] **Окно Pending Ops в настройках** (0.8.3): размер защитного буфера (15 мин / 5 мин / 1 мин / 10 сек / Выключено) выбирается в разделе «Trakt». По умолчанию остаётся 15 мин — поведение не меняется. Меньшее значение даёт быстрее проверять разные сценарии без ожидания. «Выключено» полностью обходит буфер: ничего не сохраняется, на diff ничего не накладывается, синхронизация работает «как есть».
- [x] **Watchlist: фикс дубля movie+show с одним tmdb id** (0.8.4): до 0.8.3 add слался в обе корзины Trakt (`movies` и `shows`), потому что считалось, что неподходящий тип уйдёт в not_found. На практике TMDB id 124364 одновременно занят и фильмом «Dangerous Obsession», и реальным сериалом пользователя — Trakt добавлял обе сущности, а наш собственный dedup по tmdb id маскировал пару в diff'е. Теперь тип резолвится до отправки через `resolveCardType` (поля карточки → fallback `/search/tmdb/:id`), шлём только корректный массив.
- [x] **Взаимное исключение статусных папок** (0.8.6): починили баг с двойным статусом — пользователь вручную помечал «Просмотрено» сериал, который Trakt ещё не успел переклассифицировать в continued, и карточка оставалась одновременно в Просмотрено и в Продолжение следует. Причина: per-folder `Favorite.remove(folder, card)` фаерил listener, но карточка возвращалась в папку между итерациями (видно в логе — никаких `Favorite.add` событий между двумя последовательными `remove`). Решение скопировали из `trakt_by_lampame`: перед расстановкой mark'ов вызываем `Lampa.Favorite.check(card)` и принудительно снимаем mark'и с конфликтующих папок (look/viewed/continued взаимно исключающие). После Promise.all per-folder синхронизаций — финальный verify-проход для страховки.
- [x] **Двойной статус — диагностика + real-card lookup** (0.8.7): 0.8.6 фикс не сработал — лог Running Point показал, что `enforceStatusExclusivity` зовёт `Favorite.remove(viewed, stub)`, listener event приходит, `syncStatusFolder` следом удаляет карточку из массива папки (`removed: 1`), но `Favorite.check` после всего этого всё равно отдаёт `viewed: true`. То есть в Lampa.Favorite **mark и содержимое папки — разные хранилища**, и `Favorite.remove(folder, card)` чистит только массив. Чтобы снять реальный mark, нужно передать ту самую карточку, с которой Lampa положила её в storage marks (со всеми полями). С 0.8.7 ищем настоящую карточку через `findRealCard` по всем известным папкам Lampa (book/like/wath/history/look/viewed/continued/thrown/scheduled/card) и передаём её. Если не нашли — fallback на stub с `id: Number(...)` (раньше был String — несовпадение типа могло мешать match'ингу). Дополнительно одноразово дампим `Object.keys(Lampa.Favorite)`, исходник `Favorite.remove`, `Lampa.Storage.get('favorite')` — чтобы по логу следующего сценария точно понять, где живёт mark и что Lampa внутри проверяет.
- [x] **Reconcile типа в watchlist** (0.8.5): обобщили разовый cleanup из 0.8.4 в постоянную проверку. При каждом чтении watchlist для каждого id, который есть и в Trakt, и в local Lampa book, тип сверяется с `resolveCardType(localCard)`. Если в watchlist неверный тип — удаляем; если правильного типа нет — добавляем. Лечит и исторические пары movie+show, и случай «в watchlist остался только неправильный тип» (артефакт неудачного cleanup'а 0.8.4, где для определения local-типа использовался хрупкий `cardIsShow` и на тощих карточках удалялся не тот тип).
- [x] **Синхронизация отметок просмотра серий** (0.9.0 → 0.9.1): три направления — отметить серию в Lampa → отметится в Trakt (работало и раньше через scrobble в `trakt_by_lampame`), снять в Lampa → снимется в Trakt (раньше было no-op, ломалось при ручной отметке «не смотрел»), отметить/снять в Trakt → подтянется в Lampa (раньше Lampa жила своей жизнью). Trakt — источник правды: при открытии Избранного тянем `/sync/watched/shows` (TMDB→Trakt id mapping), затем для каждого сериала, у которого в Trakt есть хоть один просмотренный эпизод — `/shows/:trakt_id/progress/watched` и сверяем с `Lampa.Timeline` (storage `file_view`, ключ — `Utils.hash([season, sep, episode, original_name||original_title])`). Чего нет в Lampa, но есть в Trakt — пишем `Timeline.update({percent:100, received:true, ...})`. Чего нет в Trakt, но есть в Lampa с percent ≥ 90 — оставляем (уже отмеченное юзером не трогаем; реальная отметка прилетит в Trakt от scrobble). На стороне Lampa слушаем `Timeline.listener('update')`: percent=0 → `POST /sync/history/remove` с `{shows:[{ids:{tmdb}, seasons:[{number, episodes:[{number}]}]}]}` (фикс «снял в Lampa — осталось в Trakt»). Защита от петли — два независимых флага: `_selfWriteMark[hash]` ставит наш `markEpisodeInLampa` (listener percent=0 трактует как echo и игнорирует), `_pendingLampaMark[hash]` ставит listener при percent ≥ 90 (используется только в reconcile, чтобы не снять отметку до того, как chuck-плагин допишет её в Trakt). Сделано простым pull-reconcile (а не realtime push): по требованию пользователя — «менее нагрузочно для системы, отметки серий не суперважная штука, потеря раз в месяц не страшна». В 0.9.0 было два бага, исправлены в 0.9.1: (1) `_pendingLampaMark` использовался и в listener, и в reconcile, и при тапе по серии «отметил → percent=95 → ставит pending → тап «снял» → percent=0 → блокируется тем же pending'ом» — listener не пускал legitimate user-unmark в Trakt. Разделил на два флага. (2) `/shows/:id/progress/watched` ожидает Trakt id, а не TMDB. С TMDB id Trakt отдавал прогресс случайного сериала с тем же числом — `e.completed` везде false, ничего не помечалось. Резолвим TMDB→Trakt через mapping из `/sync/watched/shows`, шоу без watched в Trakt пропускаем (там нечего синхронизировать).
- [x] **Папка Брошено ↔ кастомный Trakt user list** (0.10.0 → 0.10.1 → 0.11.0): двусторонняя синхронизация Lampa `thrown` ↔ Trakt. Прошли через два неудачных подхода: 0.10.0 использовал `/users/hidden/dropped` (родная Trakt-кнопка «Drop Show», полностью прячет из Continue Watching) — пользователь хотел эффект Moviebase «Stop Watching» (карточка серая + Restore), а Drop его не даёт. 0.10.1 переключили на legacy `/users/hidden/progress_watched` — на тестах не дал видимого эффекта (запрос либо не уходил, либо Trakt не реагировал). С 0.11.0 переехали на принципиально другую модель: thrown ↔ кастомный Trakt user list `Брошено`. Преимущества: (1) лист поддерживает и фильмы, и сериалы — `hidden/dropped` и `hidden/progress_watched` shows-only; (2) карточка из листа в поиске/каталоге Trakt и Moviebase получает значок папки на постере — единственный из вариантов, где «брошенность» видна вне CW; (3) лист — наша собственность, не зашумлён чужими интеграциями. Параллельно при add/remove дёргаем `/users/hidden/dropped` как helper для очистки CW (только для сериалов — фильмам в CW делать нечего); если helper упал, основной эффект сохраняется через лист. Slug листа кешируется в `Lampa.Storage` чтобы не ходить в `/users/me/lists` каждый цикл; `findOrCreateThrownList` ищет лист по имени и создаёт его при первом write если его нет. Read-путь: `GET /users/me/lists/:slug/items?limit=500` параллельно с классификацией; обе ветки (`it.show` и `it.movie`) добавляются в `desired.thrown`. Write-путь: `POST /users/me/lists/:slug/items[/remove]` + (для шоу) `POST /users/hidden/dropped[/remove]`. Тип карточки резолвим через `resolveCardType` (тощая карточка от листенера часто без полей). `STATUS_FOLDERS` расширен с `['look','viewed','continued']` до `['look','viewed','continued','thrown']` — параметрические `enforceStatusExclusivity`/`verifyStatusFoldersClean`/`applyStatusPendingOps` дают эксклюзивность автоматически. История просмотров и watched-state не трогаются — лист их не знает.

Вне скоупа (решено): Нравится, Позже, Запланировано, История просмотров, оценки.
