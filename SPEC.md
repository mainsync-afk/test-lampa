# SPEC — trakt_folder_sync

Спецификация поведения плагина. Пишется до кода. Язык — человеческий, не технический.

## Философия

Lampa — клиент Trakt. **Trakt — единственный источник правды.** Любое действие пользователя в Lampa транслируется в вызов Trakt API; Lampa никогда не хранит собственного состояния, которое может разойтись с Trakt. На каждой синхронизации Lampa зеркалит то, что сейчас в Trakt.

Последствие: петель «Lampa думает одно, Trakt другое» в принципе не возникает. Если Trakt отдал состояние — Lampa его показала, и точка.

## Категории Lampa

В Lampa два типа папок в Избранном:

- **Теги** — несколько одновременно: Закладки (book), Нравится, Позже, История просмотров.
- **Статусы** — только один: Смотрю (look), Просмотрено (viewed), Запланировано, Продолжение следует (continued), Брошено (thrown).

Плагин работает только с: **book, look, viewed, continued**.

## Маппинг Trakt → Lampa

| Lampa | Trakt | Направление |
|---|---|---|
| Закладки (book) | Watchlist (фильмы + сериалы) | Двусторонняя |
| Смотрю (look) | Сериалы с `completed > 0 && aired > completed` | Trakt → Lampa |
| Просмотрено (viewed) | Фильм в истории / сериал полностью досмотрен и закрыт | Trakt → Lampa (производная) |
| Продолжение следует (continued) | Сериал догнан, но ещё выходит | Trakt → Lampa (производная) |

## Логика статусов сериала

Берём из Trakt:
- `show.status` (поле объекта шоу при `extended=full`) — одно из: `returning series`, `in production`, `planned`, `canceled`, `ended`.
- `progress.aired` — сколько эпизодов вышло.
- `progress.completed` — сколько из них просмотрено пользователем.

Правила:

```
completed == 0
  → сериал не смотрели. В look/viewed/continued не попадает.
  → если в watchlist — остаётся только в Закладках.

completed > 0 && aired > completed
  → есть непросмотренные вышедшие эпизоды → Смотрю.

completed > 0 && aired == completed && status ∈ {ended, canceled}
  → сериал завершён и полностью досмотрен → Просмотрено.

completed > 0 && aired == completed && status ∈ {returning series, in production, planned}
  → пользователь догнал, ждёт новые серии → Продолжение следует.
```

Важно: «completed > 0» означает, что **хотя бы один эпизод отмечен в Trakt как просмотренный**. В Trakt это происходит при scrobble'е при воспроизведении либо при ручной отметке через API. Простое открытие карточки в Lampa это не триггерит.

## Логика статусов фильма

Фильм попадает в **Просмотрено**, если он есть в истории Trakt (`/sync/history`, либо `plays > 0` у фильма в watched-списке).

Фильм в **Смотрю/Продолжение следует** не попадает никогда — эти статусы только для сериалов.

## Ручные действия пользователя

Принцип: **действие пользователя в Lampa = вызов Trakt API**. Lampa свой статус не меняет до подтверждения от Trakt (оптимистично отображает, но источник истины — Trakt после следующего чтения).

| Действие в Lampa | Вызов Trakt API |
|---|---|
| Добавить в Закладки | `POST /sync/watchlist` |
| Убрать из Закладок | `POST /sync/watchlist/remove` |
| Фильм → Просмотрено | `POST /sync/history` (добавить в историю) |
| Сериал → Просмотрено | `POST /sync/history` на все вышедшие эпизоды; после этого `aired == completed`, и Trakt отдаст сериал либо как Просмотрено (если ended/canceled), либо как Продолжение следует (если ещё выходит). Пользователь увидит конечный статус после следующей синхронизации. |
| Убрать из Просмотрено | `POST /sync/history/remove` |

Ключевое: пользователь не «переносит» сериал в Просмотрено — он создаёт **условия**, при которых Trakt отдаст правильный статус. Если сериал ещё выходит, он окажется в Продолжение следует, а не в Просмотрено — это корректно.

## Триггеры синхронизации

- **При открытии экрана Избранного** — полный проход: читаем watchlist + прогрессы, пересчитываем book/look/viewed/continued.
- Ручных кнопок синхронизации не нужно (thrown не реализуется).

## Зависимости и границы с `trakt_by_lampame`

Плагин `trakt_by_lampame` остаётся зависимостью, но используется **ограниченно**:

**Берём из него:**
- OAuth: авторизация, хранение токена в `Lampa.Storage.get('trakt_token')`, обновление по refresh-токену.
- Scrobbling эпизодов во время воспроизведения.
- UI авторизации в настройках (кнопка «Войти в Trakt» и т.п.).

**НЕ берём из него:**
- `window.TraktTV.api.upnext()` — использует VIP-эндпоинт `/sync/progress/up_next_nitro`, отдаёт некорректные данные. Вместо этого сами вызываем публичные эндпоинты `/sync/watched/shows` и/или `/users/me/progress/watched/activity?hide_completed=true`.
- В перспективе — и остальные API-методы (`watchlist`, `addToWatchlist` и т.п.) переводим на прямые `fetch` к `api.trakt.tv` с токеном из Lampa.Storage. Это убирает зависимость от интерпретации данных чужим плагином.

**Параллельная работа:** `trakt_by_lampame` продолжает подгружать данные для своих home-рядов («Trakt Up Next», «Trakt Watchlist» и т.п.) при открытии главного экрана Lampa. Наш плагин читает независимо при открытии Избранного. Пересечений в запросах на запись нет, rate limit Trakt (1000/5мин) запасает на порядок. Если home-ряды не нужны — отключаются в настройках главного экрана Lampa, OAuth при этом продолжает работать.

## Задержки Trakt API

Trakt API может отдавать устаревшие данные 5–15 минут после записи (интерфейс Trakt и API часто рассинхронизированы).

**Следствия:**

- После write-запроса к Trakt (например, `addToWatchlist`) **не перечитываем** то же самое сразу — получим старое состояние и решим, что запись не прошла.
- После write'а считаем, что Trakt принял, если пришёл 2xx. Локально отображаем ожидаемый результат. Следующая полная синхронизация выровняет.
- Опционально (позже): список «pending operations» за последние N минут, которые игнорируются при сверке. Пока без этого.

## Что НЕ реализуется

Решено 2026-04-24, вынесено за скоуп:

- **Брошено (thrown)** — отложено до изучения логики «drop show» в Moviebase. Пока никак не мапится.
- **Нравится, Позже, Запланировано, История просмотров** — не трогаем.
- **Оценки/ratings** — не синхронизируются.
- **Поля `reset_at`, `hidden`** из progress — пока игнорируем. Базовую логику не меняют.

## Глоссарий

- **watchlist** (Trakt) — список «хочу посмотреть», без различия фильм/сериал.
- **up-next** (Trakt) — сериалы, у которых есть следующий непросмотренный эпизод.
- **progress** (Trakt) — объект с `aired`, `completed`, `next_episode` и т.п. для конкретного сериала.
- **scrobble** (Trakt) — событие start/pause/stop во время воспроизведения, которое автоматически помечает эпизод просмотренным.
